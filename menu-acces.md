## 🏛️ Hiérarchie des rôles

### 🔷 Niveau 1 — Stratégique (système)
| Rôle | Périmètre |
|---|---|
| **GLOBAL_CONTROL** | Super administrateur stratégique. 🔓 Accès total à **tous** les modules. |
| **SYSTEM_ADMIN** | Administrateur technique. 🔓 Tous modules, Users, Settings, System Config. |
| **BODY_CONTROL** | 👁 Lecture seule sur **tous** les modules. |

### 🔷 Niveau 2 — Direction
| Rôle | Périmètre |
|---|---|
| **DIRECTOR** | Vision globale de l'établissement. CRUD sur Students/Calendar/Pedagogy/Reports/Profile. 👁 sur le reste. ❌ sur la config technique. |
| **ACADEMIC_PRINCIPAL** | Direction pédagogique. CRUD sur Students/Attendance/Calendar/Timetable/Announcement/Pedagogy/Course tracking/Reports/Profile. 👁 sur le reste. |
| **DISCIPLINE_PRINCIPAL** | Direction discipline. CRUD sur Students/Attendance/Behavior/Calendar/Timetable/Announcement/Reports/Profile. ❌ sur le reste. |

### 🔷 Niveau 3 — Opérationnel
| Rôle | Périmètre |
|---|---|
| **RECEPTIONIST** | Accueil. CRUD sur Students/Announcement/Profile. 👁 Finance (factures)/Reports basic/Calendar. |
| **ACCOUNTANT** | Comptabilité. CRUD sur Finance/Reports finance/Profile. 👁 sur Students/Announcement/Calendar. |
| **HR** | Ressources humaines. CRUD sur Employees/Reports HR/Profile. 👁 sur Announcement/Calendar. |
| **DRIVER** | Chauffeur. 👁 Transport (route assignée)/Announcement/Calendar. 🔓 Profile. |
| **TEACHER** | Enseignant. CRUD sur Pedagogy (classes assignées)/Attendance/Profile. 👁 sur Behavior/Course tracking/Students (ses classes)/Reports/Announcement/Calendar. |

### 🔷 Niveau 4 — Utilisateurs finaux
| Rôle | Périmètre |
|---|---|
| **STUDENT_PARENT** | Parent. 👁 Students/Finance (factures & paiements)/Announcement. ❌ Profile (⚠️ à confirmer — à ajouter si accès au profil famille requis). |
| **STUDENT** | Élève. 👁 Students (propre dossier)/Announcement/Calendar. 🔓 Profile. |
| **BOARDING** | Responsable internat. 🔓 Boarding. 👁 Announcement/Calendar. 🔓 Profile. |
| **DAYCARE** | Responsable garderie. 🔓 Daycare. 👁 Announcement/Calendar. 🔓 Profile. |
| **RESTAURANT** | Responsable restauration. 🔓 Restaurant. 👁 Announcement/Calendar. 🔓 Profile. |
| **STORAGE** | Magasinier. 🔓 Storage. 👁 Announcement/Calendar. 🔓 Profile. |

### 🔷 Sans accès
| Rôle | Périmètre |
|---|---|
| **NONE** | ❌ Aucun accès. Compte désactivé / suspendu. |

## ⚠️ Points à valider

1. **STUDENT_PARENT → Profile** : actuellement `❌`. Logiquement un parent devrait pouvoir gérer son profil et celui de ses enfants. À confirmer.
2. **DIRECTOR → Logistics** : non listé dans la matrice originale (présent pour ACADEMIC_PRINCIPAL 👁). Faut-il l'ajouter en `👁` pour la direction ?
3. **BODY_CONTROL** : rôle d'audit/contrôle. Confirmer qu'il n'a aucune action d'écriture, même sur des cas sensibles.
4. **Teacher sur Timetable** : l'enseignant consulte-t-il le timetable de ses classes ? (non listé actuellement).
5. **DRIVER sur Reports** : pas d'accès aux reports de transport. Est-ce volontaire ?
